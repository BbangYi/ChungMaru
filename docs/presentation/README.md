# 청마루 발표자료 작업 폴더

이 폴더는 발표 자료를 만들기 위한 source, 계획, 근거 요약, 생성 산출물을 한곳에서 관리합니다.

## 폴더 구조

| 경로 | 역할 | Git 추적 |
| --- | --- | --- |
| `청마루_발표자료_v1.source.md` | 발표 흐름, 슬라이드 문장, 발표자 노트의 기준 파일 | 추적 |
| `final-ppt-generation-brief.md` | 다른 AI에 전달할 최종 PPT 생성용 브리프 | 추적 |
| `timeline-ppt-generation-brief.md` | 타임라인형 보조 브리프 | 추적 |
| `prmopt.md` | 기존 일반형 PPT 생성 브리프 | 추적 |
| `plan.md` | 발표 자료 구성 계획, 검증 기준, 후속 작업 | 추적 |
| `materials/` | 발표에 필요한 요약 근거와 분석 자료 | 추적 |
| `ppt-assets/` | PPT에 바로 넣을 그래프, 도식, 기존 PPTX 추출 media | 추적 |
| `*.pdf`, `*.pptx` | 발표용 내보내기 결과물 | 추적 제외 |

## 관리 원칙

- 발표 내용의 source of truth는 Markdown 파일로 둡니다.
- 단순 PPT를 외부 AI로 만들 때는 `final-ppt-generation-brief.md`를 우선 사용하고, 세부 발표자 노트가 필요할 때만 `청마루_발표자료_v1.source.md`를 함께 제공합니다.
- 그래프, 구조도, 개발 현황표 같은 이미지 자료는 `ppt-assets/`에 모아두고, PPTX/PDF 산출물 자체와 분리해 관리합니다.
- 최종 발표 순서는 `표지 -> 개발 계획 -> 개발 진행상황 -> 완료된 산출 지점 -> 모델 평가 -> 제약/어려움 -> 앞으로의 계획`을 우선합니다.
- NotebookLM식 큰 제목/강한 카드 구성과 타임라인 중심 구성은 보조안으로만 봅니다.
- PDF/PPTX/HWPX/PNG/대용량 JSON은 재생성 가능한 산출물로 보고 Git 추적에서 제외합니다.
- 원본 로그와 대용량 분석 파일을 그대로 보존하기보다, 보고서에 쓸 수 있는 실패 원인, 제약, 선택한 해결책, 검증 결과를 작은 문서로 남깁니다.
- Android 실시간 탐지 개선 내용은 실제 검증 결과가 생길 때만 발표 자료에 반영합니다.

## 현재 발표 메시지

청마루는 단순 욕설 마스킹 도구가 아니라, Chrome DOM과 Android 접근성/OCR처럼 서로 다른 플랫폼 수집 경로에서 텍스트를 모으고 backend 모델의 문맥 판정과 evidence span을 이용해 화면에 안전하게 반영하는 유해 텍스트 중재 시스템입니다.
